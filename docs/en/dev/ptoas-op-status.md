# PTOAS Op Status Matrix

Rows = **all ops listed in PTOAS docs** (148 unique ops after de-dup).
Column status is a snapshot checked against **pypto source**
(last status audit: 2026-06-23). **When you add or change an op, update the matching row.**

> Keep EN and zh-CN rows in sync after any op/ST change.

## Completion criteria

**An op is "complete" only if it has ST coverage.** Even if codegen is wrong,
an ST will catch it on hardware; **ops without ST are incomplete**
(regardless of frontend/codegen). Therefore in this table:

- `pypto frontend✅ + ST❌` → **incomplete** (frontend exists but not hardware-validated;
  codegen may be wrong, or a known a2a3 ISA/cube/ptoas defect caused the ST to be
  pulled — both count as incomplete).
- `ST✅` → covered by hardware ST, treated as complete.

> **About 148 vs 150**: PTOAS docs report "150 total" as row count, including 2
> duplicate rows — `pto.tstore` and `pto.tmov` each appear twice (base row +
> attribute-modeling detail row marked "covered by base row"). This table
> de-duplicates to 148 unique ops with no omissions.

## Legend

- **Level**: registration level in pypto (tile / tensor / tile+tensor / comm=distributed)
- **PTOAS API**: ✅ = PTOAS has a corresponding interface (pto-isa instruction); ❌ = not implemented / ISA_ONLY
- **pypto-tile / -tensor frontend**: ✅ = `REGISTER_OP("tile|tensor.<op>")` registered;
  `—` = N/A (comm ops are not tile/tensor-level; provided by distributed/system APIs)
- **ST**: ✅ = `tests/st/` has a real ST that directly exercises the op; `—` = for comm, see next column
- **distributed ST**: comm ops only — ✅+evidence file = covered under `tests/st/distributed/`; ❌ = none; non-comm = `—`
- **Notes**: `NEW`=added in this wave (PR #1824); `MISSING`=in PTOAS but not yet in pypto (see add-op skill);
  `codegen-incomplete`=IR/transforms exist but no codegen; `FP variant`/`internal`/`distributed`=variant/internal/distributed

> universe + name map embedded in the generator, temporarily under `temp/`). Day-to-day
> updates are **manual per row**: after adding an op, flip that row's frontend/ST columns to ✅.
